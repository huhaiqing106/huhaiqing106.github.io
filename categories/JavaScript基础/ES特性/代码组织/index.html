生成器

  1. 不要把yield语句放在finally子句内部
  2. return方法会自动调用比如for..of
  3. throw方法不会被自动调用，但是可以被try..catch

模块

旧方法

  • 异步模块定义AMD
  • 通用模块定义UMD

ES6模块

  • 1、ES6使用基于文件的模块，一个文件一个模块。
  • 2、ES6模块的API是静态的。
  • 3、ES6模块是单例。
  • 4、模块的公开API中暴露的属性和方法并不是普通的值或引用的赋值。
  • 5、导入模块和静态请求加载这个模块是一样的。

新方法

  • export

    • 命名导出

    • 默认导出

      • export default .. 接收一个表达式
      • export default function foo… 默认导出绑定实际上绑定的是标识符而不是它的值
      • 不需要更新引用模块的值用export default…，需要更新引用值用export{… as default}

  • import

    • 命名空间导入

      • import * as xx from ‘xxx’

    • import的结果声明会“提升”

类

class

  • 类方法是不可枚举的，而对象方法默认是可以枚举的。

  • class foo 与 function Foo() 区别

    • 1、由于前ES6可用的Foo.call(obj)不能工作，class Foo的Foo()调用必须通过new来实现。
    • 2、function foo 是“提升的”，class foo并不是，extends …语句指定了一个不能“提升”的表达式
    • 3、全局作用域中的class Foo创建了这个作用域的一个词法标识符Foo，但是function Foo并没有创建同名的全局对象属性

extends和super

  • 用于两个函数原型之间建立[[Prototype]]委托链接
  • super是静态绑定的，后期无法通过call、bind、apply修改
  • class和extend可以构建内置类的子类

new.target

  • new.target 总是指向new实际上直接调用的构造器
  • 如果new.target是undefined，就可以得知这个函数不是通过new调用的

static

  • static方法是直接添加到这个类的函数对象上的，而不是在这个函数对象的prototype对象上
  • static方法在函数构造器之间的双向/并行链上

Symbol.species

  • static 适用的一个地方就是为子类设定Symbol.species getter
  • 如果当任何父类方法需要构建一个新实例，但是不想使用子类的构造器本身时，这个功能使得子类可以通知父类应该使用哪个构造器

迭代器

  1. Symbol.iterator会产生一个新的迭代器
  2. next() 迭代
  3. return() thorw() 可选
