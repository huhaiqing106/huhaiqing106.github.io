Promise

  1. Promise是事件循环机制
  2. Promise只能被决议（完成或者拒绝）一次
  3. Promise永远在队列尾部添加微观任务，setTimeout等宿主API，则会添加宏观任务
  4. reject后面的东西一定会进入then中的第二个回调，如果没有第二个回调，则进入catch；但是网络异常会直接进入catch不会进入第二个回调

Thenable

任何提供了the(…)函数的对象（或函数）都被认为是thenable，可以使用Promise机制解释

Promise API

  1. Promise.resolve(…) 是thenable的解决方案，它会“封装”成一个真正的Promise

  2. Promise.all([…])接受一个或多个值的数组（比如：立即值、promise、thenable）

  3. Promise.all([…])等待所有都完成（或者第一个被拒绝）

    Promise.all([])将会立即完成

  4. Promise.race([…])等待第一个完成或者拒绝

    Promise.race([])将会永远挂起
