测试工具

  • Audit 工具获得网站的多个指标的性能报告
  • Performance 工具了解网站的性能瓶颈
  • Performance API 具体测量时间

V8引擎

  • Ignition => Bytecode

  • TurboFan => Machine Code

    执行效率优于Bytecode

  • Lazy-Compile

    • 函数没执行前会先预解析，到执行才会被解析编译
    • 可以通过把函数添加到括号中解决预解析

  • 优化策略

    • 采用减少代码文件的大小或者减少书写嵌套函数的方式减少编译时间
    • 我们应该尽可能保证传入参数的类型一致，让V8优化代码

碎片化优化

  • 图片优化

    • 减少像素点
    • 减少每个像素点能够显示的颜色

  • 图片加载优化

    • 不用图片，修饰图片尽量用CSS替代
    • 移动端，计算出适配屏的高度，再请求CDN对应裁剪好的图片
    • 小图用base64格式
    • 多图合并到一张（雪碧图）
    • 选择正确的图片格式
      • 能显示webP格式的游览器用webP格式，因为更好的图像数据压缩算法、更小的体积，但兼容性不好
      • 小图用PNG，尽量SVG
      • 照片用JEPG

  • DNS预解析

  • 节流

    滚动场景；不需要滚动过程中一直发起请求

  • 防抖

    重复点击场景；当用户点击一段时间后，没有再次点击就发起请求

  • 预加载

  • 懒执行

  • CDN
